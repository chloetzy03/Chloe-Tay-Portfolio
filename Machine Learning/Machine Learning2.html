Introduction and Universal Workflow

This project follows the universal workflow of Deep Learning with Python (DLWP) 4.5 to build a text classification model using the Reuters dataset. The dataset I chose consists of news articles categorized into 46 topics, making it suitable for multi-class classification. The primary objective is to compare different neural network architectures:

  • A baseline model to establish initial performance.
  • An overfitting model to analyze the impact of increasing complexity.
  • A regularized model to improve generalization.
  • Hyperparameter tuning to optimize performance.

The Universal Workflow of Machine Learning

I will be following the universal workflow of machine learning as described in DLWP 4.5 (1st edition):

1. Define the problem and assemble the dataset

  • Classify Reuters news articles into multiple topics
  • The Reuters dataset, loaded via keras.datasets.reuters.

2. Choosing metrics of success

  • Primary metric: Accuracy
  • Alternative metrics: Precision, recall, F1-score

3. Choosing an evaluation protocol

  • Split dataset into:
    • Training set
    • Validation set
    • Test set (held-out for final evaluation)

4. Prepare the data

  • Convert raw text sequences into numerical feature vectors using TF-IDF representation.
  • One-hot encode the labels.

5. Building a baseline model

  • Use a simple Dense network as the baseline model.
  • Train and evaluate its initial performance.

6. Scaling up for a model that overfits

  • Increase the number of neurons and layers to observe overfitting.

7. Regularize the model and tune hyperparameters

  • Add Dropout layers to reduce overfitting.
  • Optimize hyperparameters (e.g., learning rate, neurons per layer).

Multi-Class Classification with the Reuters Dataset

Text classification tasks like Reuters can be processed using various techniques:

  • TF-IDF Vectorization: Converts text data into numerical feature representations by assigning importance scores to words based on their frequency in the dataset.
  • Word Index Sequences: Represent each article as a sequence of word indices (pre-tokenized).
  • One-Hot Encoding of Labels: Convert categorical labels into a numerical format.
  • Neural Networks (Dense Layers): Build a feedforward model using Dense and Dropout layers.

I will be using TF-IDF vectorization to transform the input text into numerical feature vectors.

Measure of Success

I will evaluate my model using:

  • Accuracy (Baseline performance)
  • Macro and Weighted F1-Score (To assess class imbalance and overall classification performance)
  • Loss Curves (To detect overfitting)

Since this is a multi-class classification task, categorical cross-entropy will be used as the loss function.

2. Choose a Measure of Success

For a classification problem (in this case the Reuters dataset), where we are predicting news categories, the most appropriate measure of success depends on:

  1. Accuracy Useful if classes are balanced.
  2. Precision, Recall, and F1-score Useful for imbalanced datasets like Reuters.

Since the Reuters dataset is imbalanced, relying solely on accuracy might be misleading. Thus, I will use:

  • F1-score as it balances precision and recall, making it suitable for imbalanced data like Reuters.
  • Precision and Recall which helps analyze the trade-off between false positives and false negatives.
  • Categorical accuracy which is tracked as a metric during training using the categorical cross-entropy loss function.

3. Choose an Evaluation Protocol

To ensure reliable model evaluation, I will use holdout valdiation, splitting the training dataset into 80% training and 20% validation. Instead of using raw word indices, I preprocess the text using TF-IDF vectorization, which converts the text into numerical feature vectors based on word frequency importance. This method allows the model to capture meaningful relationships between words rather than treating them as isolated entities. I will also track categorical accuracy on both the training and validation sets during model training to monitor model performance.

5. Develop a Baseline Model

For the Reuters dataset, I will implement a neural network with:

  • Two hidden layers using Dense with ReLU activation to improve feature extraction and learning.
  • Softmax activation in output layer to predict probabilities across the 46 news categories.
  • Categorical Cross-Entropy Loss as the labels are one-hot encoded for multi-class classification.
  • Adam optimizer to ensure stable weight updates and better convergence.
  • Trained for 30 epochs with a batch size of 32 to allow the model more time to learn patters.

6. Scale Up: Develop a Model That Overfits

The baseline model showed strong performance in terms of accuracy (79%) but struggled with class imbalance, as seen in the low macro F1-score (54%). This suggests that frequent categories were learned well, while minor categories were poorly predicted. To investigate how the model responds to increased complexity, I created a larger model with:

  • More layers: Increased from 2 hidden layers to 3 hidden layers.
  • More neurons: Increased the number of neurons per layer.
  • More training time: 50 epochs instead of 30.
  • Lower learning rate: Reduced to 0.0005 to allow gradual weight updates.
In [14]:
from tensorflow.keras import models, layers, optimizers
import matplotlib.pyplot as plt

# Define a larger model with more layers and neurons
overfit_model = models.Sequential([
    layers.Dense(256, activation='relu', input_shape=(x_train.shape[1],)),  # Increased neuron count

    layers.Dense(64, activation='relu'),  # Another dense layer for more complexity

# Compile the model
overfit_model.compile(

overfit_model.summary()

history_overfit = overfit_model.fit(

    epochs=50,  # More epochs to let the model overfit

# HYPERPARAMETER TUNING

def build_model_tuner(hp):
#Function to define tunable model
    model = keras.Sequential()
    
    # Choose number of layers (1-3)
    n_layers = hp.Int('n_layers', 1, 3)
    for i in range(n_layers):
        units = hp.Choice(f'units_{i}', [64, 128, 256])  # Test different units
        l2_factor = hp.Float(f'l2_{i}', 1e-5, 1e-3, sampling='log')  # Adjust L2 factor
        model.add(layers.Dense(units, activation='relu', kernel_regularizer=regularizers.l2(l2_factor),
                               input_shape=(x_train_array.shape[1],) if i == 0 else ()))
        drop = hp.Float(f'drop_{i}', 0.1, 0.5, step=0.1)  # Reduce minimum dropout
        model.add(layers.Dropout(drop))
    
    # Output layer
    model.add(layers.Dense(46, activation='softmax'))
    
    # Choose learning rate
    lr = hp.Float('lr', 1e-4, 1e-2, sampling='log')
    model.compile(
        optimizer=keras.optimizers.Adam(learning_rate=lr),
        loss='categorical_crossentropy',
        metrics=['accuracy']
    )
    return model

# Define Hyperband Tuner
